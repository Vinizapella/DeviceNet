# 🔌 DeviceNet — Aplicação Real para Automação Industrial

> **Projeto desenvolvido como exemplo de aplicação real para a Unidade Curricular de Automação Industrial.**

---

## 📋 Sobre o Projeto

Este repositório contém uma implementação prática do protocolo **DeviceNet**, desenvolvida com fins educacionais no contexto da UC de **Automação Industrial**. O objetivo é demonstrar, de forma aplicada, como funciona a comunicação entre dispositivos industriais utilizando uma rede de campo baseada no padrão DeviceNet (CAN-based fieldbus).

O projeto serve como referência real de como os conceitos teóricos abordados em sala de aula se traduzem em sistemas de automação funcionais, integrando sensores, atuadores e controladores em uma rede industrial.

---

## 🏭 Contexto — UC de Automação Industrial

A disciplina de **Automação Industrial** aborda os fundamentos e tecnologias utilizadas na modernização e otimização de processos industriais. Entre os temas estudados estão:

- Redes de campo (*fieldbuses*) industriais
- Protocolos de comunicação industrial (DeviceNet, Profibus, Modbus, etc.)
- Integração entre CLPs, sensores e atuadores
- Topologias de redes industriais
- Diagnóstico e monitoramento de dispositivos em rede

Este projeto representa uma **aplicação concreta** desses conceitos, com foco no protocolo DeviceNet.

---

## 🌐 O que é DeviceNet?

**DeviceNet** é um protocolo de comunicação industrial baseado no barramento **CAN (Controller Area Network)**, amplamente utilizado para conectar dispositivos de automação como:

- CLPs (Controladores Lógicos Programáveis)
- Inversores de frequência
- Sensores e atuadores inteligentes
- Painéis de operação

### Características principais:
| Característica | Valor |
|---|---|
| Velocidade | 125 kbps / 250 kbps / 500 kbps |
| Topologia | Barramento com derivações (trunk-drop) |
| Nós máximos | 64 dispositivos |
| Protocolo base | CAN (ISO 11898) |
| Camada de aplicação | CIP (Common Industrial Protocol) |
